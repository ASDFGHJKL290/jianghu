### F-01：情感计算引擎

**需求描述**：LLM 推断对话情感，输出结构化 `emotion_state`。

**情感标签集**（共 10 种）：

| 标签 | 中文含义 | 典型触发场景 |
|------|---------|------------|
| `happy` | 开心 | 被夸奖、得到认可 |
| `sad` | 悲伤 | 提及损失、离别 |
| `angry` | 愤怒 | 被冒犯、威胁 |
| `fearful` | 恐惧 | 危险、未知威胁 |
| `surprised` | 惊讶 | 突发事件 |
| `disgusted` | 厌恶 | 不屑、嫌弃 |
| `excited` | 兴奋 | 战斗、名剑、好消息 |
| `worried` | 担忧 | 提及危险人物、未来 |
| `proud` | 自豪 | 谈及门派成就 |
| `neutral` | 中性 | 一般性问答 |

**emotion_state 数据结构**：

| 字段 | 类型 | 范围 | 说明 |
|------|------|------|------|
| `primary` | string | 情感标签集 | 主导情感 |
| `secondary` | string \| null | 同上 | 次要情感（可选） |
| `intensity` | float | 0.0~1.0 | 情感强度 |
| `valence` | float | -1.0~1.0 | 正负效价 |
| `arousal` | float | 0.0~1.0 | 唤醒度 |
| `dominance` | float | 0.0~1.0 | 控制感 |
| `trend` | string | increasing/stable/decreasing | 变化趋势 |
| `trigger_word` | string | — | 触发词 |

**情感变化规则**：
- 连续同类型情感：强度累加，上限 1.0
- 情感切换：旧强度衰减 -0.3
- 静默 >60s：强度 → 0，恢复 neutral

**LLM Prompt 扩展片段**（追加到 chat() system_prompt）：
```
【情感标注规则】（必须严格遵守）
根据回复内容选择最合适的情感标签：happy, sad, angry, fearful,
surprised, disgusted, excited, worried, proud, neutral

强度规则：
- 激动词（震惊、绝世、天下无敌）→ intensity >= 0.8
- 一般情感词 → 0.4~0.7
- 平淡叙述 → intensity < 0.4

三维情感值（PAD）：
- valence: 正面积极=正数，负面消极=负数
- arousal: 激烈兴奋=高值，平静低沉=低值
- dominance: 自信主导=高值，被动迷茫=低值

回复末尾必须以如下JSON格式输出情感标注（不得省略）：
{"emotion": {"primary":"标签","secondary":"标签或null","intensity":0.0~1.0,
"valence":-1.0~1.0,"arousal":0.0~1.0,"dominance":0.0~1.0,
"trend":"increasing/stable/decreasing","trigger_word":"触发关键词"}}
```

### F-02：动作提取器

**需求描述**：从 LLM 回复中提取 1~2 个肢体动作指令。

**动作标签体系**（共 15 种）：

| 动作 | 说明 | 优先级 |
|------|------|--------|
| `idle` | 默认待机 | 兜底 |
| `listen` | 侧耳倾听 | 语义 |
| `nod` | 点头 | 语义 |
| `shake_head` | 摇头 | 语义 |
| `bow` | 拱手行礼 | 语义 |
| `wave` | 摆手 | 语义 |
| `sword_draw` | 拔剑 | 情感+语义 |
| `sword_sheathe` | 收剑 | 语义 |
| `laugh` | 笑 | 情感 |
| `sigh` | 叹气 | 情感 |
| `think` | 沉思 | 语义 |
| `look_away` | 看向别处 | 情感 |
| `slap_table` | 拍桌 | 情感 |
| `cross_arms` | 抱臂 | 情感 |
| `point` | 指向 | 语义 |

**规则**：最多返回 2 个动作，情感触发优先于语义触发。

**LLM Prompt 扩展片段**：
```
【动作提取规则】（必须严格遵守）
从动作库中选择最符合回复内容的1-2个动作标签（不得超出此列表）：
idle, listen, nod, shake_head, bow, wave, sword_draw, sword_sheathe,
laugh, sigh, think, look_away, slap_table, cross_arms, point

规则：
- 战斗/比武/剑相关 → sword_draw（情感+语义）
- 开心/幽默/讽刺 → laugh（情感）
- 认同/肯定 → nod（语义）
- 否定/拒绝 → shake_head / wave（语义）
- 愤怒/强调 → slap_table（情感）
- 悲伤/无奈 → sigh（情感）
- 问候/尊重 → bow（语义）
- 思考 → think（语义）
- 最多返回2个动作
回复末尾必须以如下JSON格式输出动作（不得省略）：
{"actions": ["action_tag1"]} 或 {"actions": ["action_tag1","action_tag2"]}
```

### F-03：VoiceParams 语音参数生成

**需求描述**：根据 `emotion_state` 自动生成 TTS 语音参数，供 Azure TTS SSML 使用。

**VoiceParams 数据结构**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `speed` | float | 语速倍率，1.0=正常 |
| `pitch` | string | 音高偏移，如 "+10%" |
| `volume` | float | 音量，1.0=正常 |
| `style` | string | 音色风格（Azure SSML style 参数） |

**情感 → 语音参数映射**：

| emotion.primary | speed | pitch | volume | style |
|----------------|-------|-------|--------|-------|
| happy | 1.05 | +5% | 1.0 | cheerful |
| sad | 0.85 | -10% | 0.9 | somber |
| angry | 1.15 | +8% | 1.1 | aggressive |
| fearful | 1.25 | +15% | 1.0 | fearful |
| surprised | 1.1 | +12% | 1.0 | surprised |
| disgusted | 0.9 | -5% | 0.9 | disgusted |
| excited | 1.25 | +10% | 1.1 | enthusiastic |
| worried | 0.85 | -5% | 0.9 | worried |
| proud | 1.0 | +3% | 1.05 | proud |
| neutral | 1.0 | 0 | 1.0 | default |

**验收标准**：
- [ ] excited → speed=1.25, pitch="+10%", style="enthusiastic"
- [ ] sad → speed=0.85, pitch="-10%", style="somber"
- [ ] fearful → speed=1.25（最快），pitch="+15%"（最高）
- [ ] 全部 10 种情感均有对应参数

### F-04：Azure TTS 语音合成（后端）

**需求描述**：后端根据 `voice_params` 调用 Azure TTS，合成 .wav 音频文件并缓存，返回 `audio_url`。

> ⚠️ **TTS 合成逻辑在后端只写一次**，Phase 2 Vue 和 Phase 4 Unity 都只调用 `audio_url` 播放。

**合成流程**：
```
emotion_state + voice_params
        ↓
构建 Azure TTS SSML（语音标记语言）
        ↓
POST Azure REST API（Bearer Token 鉴权）
        ↓
接收 .wav / .mp3 音频流
        ↓
写入 memory/audio/{npc_name}_{timestamp}.wav
        ↓
返回 audio_url = "/audio/店小二_20260420_091930.wav"
```

**Azure TTS SSML 示例**（excited 情感）：
```xml
<speak version='1.0' xmlns='http://www.w3.org/2001/10/synthesis'
       xml:lang='zh-CN'>
  <voice name='zh-CN-XiaoxiaoNeural'>
    <prosody rate='+25%' pitch='+10%' volume='+3dB'>
      壮士！华山论剑乃天下武林盛事，距今已有百年历史……
    </prosody>
  </voice>
</speak>
```

**音频缓存策略**：
```
缓存 Key = hash(npc_name + text + voice_params)
命中缓存 → 直接返回 audio_url（不调 Azure API）
未命中    → 调用 Azure TTS → 写入 .wav → 返回 audio_url
```

**FastAPI 静态文件服务**：
```python
app.mount("/audio", StaticFiles(directory="memory/audio"), name="audio")
```

**验收标准**：
- [ ] `/api/chat` 响应包含 `audio_url` 字段
- [ ] 相同 text + voice_params 第二次请求命中缓存（不扣 Azure 费用）
- [ ] Azure API 不可用时（无 Key/网络问题）→ `audio_url = null`，reply 正常返回
- [ ] 音频文件命名格式：`{npc_name}_{timestamp}.wav`

### F-05：历史记忆持久化

**需求描述**：对话历史、关系状态、世界状态持久化到 JSON 文件，跨会话保留上下文。

**文件结构**：
```
memory/
├── session_{npc}_{date}.json     # 当天短期记忆
├── relation.json                   # 亲密度状态
├── world_state.json                # 世界状态
├── longterm/
│   └── {npc}_summary.json         # 长期记忆摘要
└── audio/                          # TTS 音频缓存（F-04）
    └── 店小二_20260420_091930.wav
```

**短期记忆（session_*.json）字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `turns[].user` | string | 用户输入 |
| `turns[].assistant` | string | NPC 回复 |
| `turns[].emotion` | string | 情感标签 |
| `turns[].intensity` | float | 情感强度 |
| `turns[].relation_delta` | int | 本轮亲密度变化 |

**关系状态（relation.json）**：
```json
{
  "default": {
    "店小二": { "intimacy": 65, "last_interact": "2026-04-20" },
    "武林盟主": { "intimacy": 50, "last_interact": "2026-04-20" },
    "神秘大侠": { "intimacy": 50, "last_interact": "2026-04-19" }
  }
}
```

**亲密度影响规则**：

| 触发条件 | 亲密度变化 |
|---------|---------|
| 正常对话 | +2~+5（积极）/ -5~-10（消极/冒犯） |
| 完成任务 | +10 |
| 放弃任务 | -3 |
| 30天无互动 | -2（衰减） |

**长期记忆（longterm/{npc}_summary.json）**：
```json
{
  "npc_name": "武林盟主",
  "summary": "玩家对华山派有浓厚兴趣，询问了论剑历史。",
  "key_events": [
    { "event": "触发华山论剑任务", "date": "2026-04-20" }
  ],
  "last_updated": "2026-04-20T10:00:00"
}
```

**验收标准**：
- [ ] 重启服务后，对话历史仍然影响 NPC 回复（如"你上次说……"）
- [ ] 亲密度影响 NPC 称呼（高→"少侠"，低→"来者何人"）
- [ ] 记忆文件损坏时自动创建空白文件，不崩溃
- [ ] `memory_update` 在 `/api/chat` 响应中返回

### F-17：NPC 状态机

**时序**：表情/动作先行（~300ms）→ 语音播放 → 表情衰减 → idle。

```
[Idle]
    │
    │ ChatClient 收到 packet
    ▼
[PreProcess]
    ├─ EmotionDriver.ApplyEmotion()     ──┐
    └─ ActionController.Enqueue()       ──┤ 并行
                                          │
    │ ~300ms 后                           │
    ▼                                     │
[Speaking] ←── AudioPlayer.PlayURL() ─────┘（音频 URL）
    │
    │ 语音播放完成
    │
    ├─ ActionController → idle
    └─ EmotionDriver → 衰减至 neutral
    ▼
[Idle]
```

**验收标准**：
- [ ] 表情/动作在语音播放前 0.3s 开始
- [ ] 语音播完 → 动作归零 → 表情衰减 → idle
- [ ] 快速连续对话时，上一轮动画被中断切到新一轮

## 七、技术约束与风险

| 约束/风险 | 说明 | 缓解方案 |
|---------|------|---------|
| Azure TTS 费用 | 按字符计费 | 缓存命中后不再调用 API |
| LLM 动作标签准确性 | 可能生成非法动作 | ActionParser 校验+过滤 |
| Blender Blendshape 数量 | 不同 SDK 版本上限不同 | 每个 NPC 至少 12 个（共用水墨风格可复用） |
| Unity 表情性能 | Blendshape 逐帧 lerp 影响 FPS | Coroutine 异步，避免 Update 逐帧 |
| 多 NPC 同时对话 | 资源竞争 | 异步 FastAPI + uvicorn 多工作进程 |
