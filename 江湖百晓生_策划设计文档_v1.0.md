## 一、项目愿景

在不改变现有对话核心的前提下，引入**人际关系系统**、**任务道具系统**，将对话从「闲聊式」升级为「江湖冒险式」。用户不只是和NPC聊天，而是真正在江湖中闯荡、交友、收集、完成任务，获得代入感与成就感。

## 二、人际关系系统

### 2.1 设计目标

NPC不再只是知识问答机器人，而是用户江湖路上的**引路人、盟友或对手**。关系越深，NPC的称呼、语气、解锁内容都会随之变化。

### 2.2 现有基础

系统已有 `MemoryManager` 和 `relation.json`，亲密度范围 0-100，分5个等级。但目前亲密度变化仅靠情感标签自动触发，比较被动。

### 2.3 关系维度（三维模型）

| 维度 | 说明 | 影响 |
|------|------|------|
| **亲密度** | 数值 0-100，反映情感亲近程度 | 称呼、语气、是否透露秘密 |
| **声望值** | 江湖正/邪声望，正负双向 | 能否触发某些任务、解锁某些NPC |
| **了解度** | NPC对用户背景的了解程度（0-5星） | NPC是否记住用户的身世、门派、承诺 |

### 2.4 亲密度等级与称呼（修订）

| 等级 | 分值区间 | 店小二称呼 | 武林盟主称呼 | 神秘大侠称呼 |
|------|----------|------------|--------------|--------------|
| 陌生人 | 0-20 | 来者何人？/客官您哪位？ | 大胆！/报上名来！ | …… |
| 初识 | 21-40 | 客官您又来啦！ | 壮士请讲 | …… |
| 相熟 | 41-60 | 少侠！/兄弟！ | 壮士 | …… |
| 信任 | 61-80 | 哎哟自己人！ | 少侠 | …… |
| 至交 | 81-100 | 兄弟！有什么事尽管吩咐！ | 少侠，本座看好你！ | 有缘。 |

**称呼触发场景：**
- 每次对话开头，NPC会用当前等级的称呼打招呼
- 关键剧情点（完成任务、拒绝请求）可临时触发特殊称呼

### 2.5 了解度机制（新增）

系统从对话中提取用户信息（身世、门派、目标、仇人），存储在 `relation.json` 的 `known_facts` 字段中。NPC只有在了解度达到一定程度时，才会触发**专属剧情对话**。

```
known_facts = {
    "origin": "少林寺弃徒",
    "goal": "夺回屠龙刀",
    "enemy": "青城派",
    "weapon": "长剑"
}
```

当了解度≥3星时，NPC可能主动提及用户背景：
> 店小二："兄弟，听说你是少林寺出来的？少林的功夫……哎，那可是真本事啊！"

### 2.6 关系变化触发规则

| 触发条件 | 亲密度变化 | 声望变化 |
|----------|-----------|----------|
| 正常闲聊 | +0~+2 | 0 |
| 完成正向任务（如帮NPC做事） | +5~+15 | +5正或负（视任务性质） |
| 拒绝NPC请求 | -3~-10 | 0 |
| 主动送礼（道具） | +10~+20 | 0 |
| 背叛/欺骗NPC | -20~-50 | -10（邪） |
| 击败对手/为NPC出头 | +5~+10 | +10（正） |
| 询问NPC身世/往事 | +2~+5 | 0 |
| 长时间不互动（>7天） | 每日-1衰减 | 0 |

## 三、任务系统（增强版）

### 3.1 现有任务结构

系统已有 `ALL_QUESTS`，按关键词触发，由NPC权限控制。

### 3.2 新任务分类（五类）

| 任务类型 | 触发方式 | 代表任务 |
|----------|----------|----------|
| **传闻任务** | 关键词触发（RAG检索+关键词双重匹配） | 初入江湖、侠客行、门派风云 |
| **委托任务** | NPC主动提出（亲密度≥40解锁） | 送信、寻物、打探消息 |
| **限时任务** | 每日重置，需当日完成 | 华山论剑每日擂主挑战 |
| **传说任务** | 多步链式任务，需前置任务完成 | 寻找九阴真经（需先完成侠客行） |
| **奇遇任务** | 随机触发（每日概率触发） | 桃花岛偶遇、荒山夜话 |

## 四、任务道具系统（新增）

### 4.1 设计目标

道具系统让江湖体验从「纯文字」升级为「可感知」——用户完成任务或探索会获得道具，道具又能触发新对话或任务，形成闭环。

### 4.2 道具体系结构

```
道具属性：
- name: 道具名称
- type: 类型（凭证/武器/书籍/消耗品/特殊）
- rarity: 稀有度（普通/珍稀/传说）
- source: 获取来源（任务奖励/商店购买/RAG触发/赠送）
- effect: 使用效果（触发剧情/提升声望/提升亲密度）
- usable: 是否可主动使用
- tradeable: 是否可交易（未来扩展）
```

## 五、UI/前端设计建议

### 5.1 道具面板（新增）

在现有任务面板旁边新增「行囊」标签页：
- 格子展示式道具列表（类似RPG道具栏）
- 道具有图标、名称、稀有度颜色（普通灰、珍稀蓝、传说金）
- 点击道具显示详情（说明、效果、来源）
- 消耗品有"使用"按钮，凭证/书籍有"使用"和"展示给NPC"

### 5.2 人际关系面板（新增）

- NPC头像旁边显示亲密度条（数值+进度条）
- 点击可展开：了解度星级、已知信息摘要、声望值
