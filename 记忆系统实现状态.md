## 一、四层架构

```
行为层  ✅ system_prompt + mood_behavior_prompt()
知识层  ✅ RAG (ChromaDB + BGE)
经历层  ✅ 六因子权重 + 固化/衰减/压力 ✅（本次完成）
社交层  ✅ relation.json
```

各层边界清晰：
- **行为层**只管"怎么说话"（语气、动作），`prompts.py`
- **知识层**只管"世界是什么"（事实），`.md → ChromaDB → k=2 检索`
- **经历层**只管"发生过什么"（具体事件），`core.py → experience JSON → 权重排序`
- **社交层**只管"跟谁关系如何"（亲密度），`relation.json`

## 二、经历层完整链路

```
① write               ② prompt inject          ③ decay
┌──────────┐         ┌──────────────┐         ┌──────────┐
│ LLM 回复  │         │ get_longterm │         │ 游戏日变更│
│ highlights│ ──→    │ 固化全量       │ ──→    │ _daily_   │
│ + memory │         │ 普通 Top 9    │         │ decay()  │
│          │         │              │         │          │
│ ↓        │         │ ↓            │         │ ↓        │
│ update_  │         │ 【最在意的事】│         │ W *= γ   │
│ longterm │         │ 【经历】      │         │          │
│          │         │              │         │          │
│ ↓        │         │ ↓            │         │ ↓        │
│ 六因子W  │         │ 注入 prompt   │         │ W<0.1+30d│
│ 固化判定 │         │              │         │ → 遗忘   │
└──────────┘         └──────────────┘         └──────────┘
```

### 2.1 写入

**每轮对话后自动触发**（`chat.py`）：

1. `MemoryParser.parse(raw)` — 从 LLM 响应的 `{"memory":"..."}` 提取长段记忆
2. `_parse_highlights(raw)` — 从 LLM 响应的 `{"highlights":["话题1","话题2"]}` 提取标签
3. 每条调用 `update_longterm(npc_name, text)` → 六因子权重 → 判定固化

**LLM 输出格式**（`prompts.py` MEMORY_RULES）：
```json
{"memory": "玩家喜欢喝酒", "highlights": ["玩家打听江湖规矩", "聊了丐帮帮主之争"]}
```

### 2.2 权重公式

`_compute_weight(npc_name, event, access_count, importance, emotion, days_old)`

```
W = 0.25·F + 0.20·I + 0.15·E + 0.20·R + 0.10·S + 0.10·X + noise
```

| 因子 | 计算 | 作用 |
|------|------|------|
| F 频次 | min(1, log2(ac+1)/log2(20)) | 反复提的事记更牢 |
| I 重要性 | 0.4~0.8（关键词匹配） | 任务/秘籍权重高 |
| E 情绪 | 0.3~0.7（情感词匹配） | 带情绪的事印象深 |
| R 近因 | 1-day_old/90 | 越近的事记忆越新 |
| S 感官 | NPC 偏好向量[事件类型] | 店小二重听觉，黄衫女重视觉 |
| X 专业 | 命中领域=1.5，否则0 | 平一指记伤病，洪七公记吃喝 |

### 2.3 固化

`_check_consolidate(evt)`，满足任一即固化（fixed=true，不再衰减）：

- `access_count ≥ 5`
- `importance × emotion > 0.8` 且 `access_count ≥ 2`

### 2.4 衰减与遗忘

`_daily_decay(evt, game_day)`：

```
日衰减率 = 0.95 + min(1, age/30) × 0.04
新记忆(当天): ×0.95 → 掉得快
30天历史: ×0.91 → 掉得慢
90天+: ×0.985 → 几乎不掉
```

遗忘条件：`W < 0.1` 且 `30天未访问` → 从存储中删除。

### 2.5 压力衰减

`_apply_pressure()` — 10 分钟内写入 > 3条/分钟 → 所有未固化记忆 ×0.96。

### 2.6 检索与注入

`get_longterm(npc_name)` → 按权重降序 → `record_references()` 更新访问记录：

- **固化记忆** → 全部注入 prompt（【最在意的事】）
- **普通记忆** → 权重前 9 条注入 prompt（【经历】）
- **未访问的记忆** → 自动衰减

### 2.7 对话窗口

`chat.py` build_chat_response：

- 最近 10 轮完整对话
- 10 轮前的话题摘要（`【之前聊过】`）
- 来自 session JSON 文件

## 三、社交层

`relation.json`：
```json
{"default": {"店小二": {"intimacy": 65}}}
```

不参与权重/衰减。由 `save_turn` 自动更新亲密度（`RELATION_DELTA` 映射）。
