## 测试二：章节进度系统

### 2.1 查看当前章节
- 页面顶栏应显示：`🏮 江 湖 百 晓 生 | 📖 第1章：初入江湖`
- F12 控制台输入以下，查看当前章节信息：
```js
// 在浏览器控制台执行
app._instance.setupState.chapterInfo
```

### 2.2 触发第一章进阶
用**新用户**（或清档后重新来），与关键 NPC 对话，输入包含关键词的消息：

| 章节 | 关键词（输入包含其中之一即可） |
|------|------------------------------|
| 第一章 | `初入江湖` `完成第一章` `选择门派` `领悟武` |

**操作**：
1. 与「店小二」或「武林盟主」对话
2. 输入：`「我已初入江湖，准备选择门派，领悟武学之道」`
3. 查看响应

**预期结果**：
- 后端返回 `chapter_advancement: {advanced: true, new_chapter: "chapter_2", ...}`
- 前端弹出 Toast：`📖 章节推进！第2章`
- 顶栏更新为：`📖 第2章：棋局已布`
- 系统消息出现在对话中：`📖 系统：章节已推进到第2章！`

### 2.3 重复触发保护
- 进阶后，再次输入同样的关键词
- 预期：`chapter_advancement.advanced === false`，不再弹通知

### 2.4 多章节连续进阶（完整流程）

清档后，依次触发各章节：

```
chapter_1 → chapter_2 → chapter_3 → chapter_4 → chapter_5 → chapter_6
```

每个章节的关键词在 `prompts.py` 的 `CHAPTER_CONFIG` 中定义。

## 常见问题

**Q：页面全乱，显示 `{{ }}` 原文**
→ 后端没启动，或从 PyCharm 63342 端口打开的。必须从 `localhost:8000` 访问。

**Q：图片全 404**
→ 后端没启动，静态文件服务不可用。启动 uvicorn 后再试。

**Q：章节不进阶**
→ 检查后端日志，`check_chapter_advancement()` 是否返回 `advanced: true`。
→ 确认 `prompts.py` 里 `CHAPTER_CONFIG` 的 `completion_keywords` 是否正确。

**Q：礼物没收到**
→ 检查 LLM 返回内容是否包含正确格式的 JSON：`{"gift": {"name": "...", "type": "...", "rarity": "..."}}`
→ 检查后端日志，`_extract_gift()` 是否成功解析。
